# Contract Builder — 只要有人社群顧問 合約產生器

## 概述

這個 skill 根據「只要有人社群顧問」過往合約範本，自動產生完整合約 .docx 檔案。合約分為兩大類型：

1. **對客戶合約**（社群合作協議 / 顧問服務合約）— 甲方是客戶、乙方是只要有人
2. **對藝人 KOL 合約**（藝人/創作者合作合約）— 甲方是只要有人、乙方是 KOL/經紀公司

產出檔案放在 `Outputs/Contract/` 資料夾。

---

## 工作流程

### Step 1：確認合約類型

先詢問使用者要產生哪一種合約：

- **A. 對客戶合約 — 社群合作協議**：只要有人承接客戶的社群經營專案
- **B. 對客戶合約 — 顧問服務合約**：只要有人提供顧問分析服務（輿情分析、策略定位等）
- **C. 對藝人 KOL 合約**：只要有人代客戶找 KOL/藝人合作

### Step 2：蒐集合約資訊

根據合約類型，用 AskUserQuestion 工具逐步收集以下資訊。一次問一到兩組相關問題，不要一次問完所有問題。

#### 所有類型都需要的基本資訊：

1. **專案名稱**（例：「農純鄉社群專案」「宏佳騰智慧電車社群專案」）
2. **甲方資訊**：公司名稱、統一編號、負責人姓名、地址
3. **乙方資訊**：
   - 對客戶合約：乙方固定為「只要有人社群顧問股份有限公司」
   - 對 KOL 合約：乙方為 KOL 的經紀公司或個人
4. **合作期間**：起始日 ~ 結束日
5. **合作費用**（未稅金額、付款方式：頭尾款比例或一次付清）

#### 對客戶合約（A/B）額外需要：

6. **服務費用頭款金額**（如有分期）
7. **銀行帳戶**（預設使用公司帳戶：永豐銀行(807)台北分行(0210)，帳號 02101800064108）
8. 若有 **KOL/藝人合作條款（第六條）**：
   - KOL 經紀公司名稱、代表的藝人名稱
   - 藝人合約簽署日期
   - 防火條款授權期間
   - 是否需要客戶也遵守防火條款（雙向約束）

#### 對 KOL 合約（C）額外需要：

6. **甲方客戶名稱**（品牌端公司名稱）
7. **乙方藝人名稱**
8. **合作日期與內容**，逐項確認：
   - 貼文數量、平台（Instagram / Facebook / YouTube / TikTok 等）
   - 上線日期
   - 限動是否贈送（幾則）
   - 是否需要導流（例：swipe up 連結）
   - 文案必帶內容（hashtag、品牌 tag 等）
   - 是否需標記客戶粉專（提供帳號）
   - 是否為共同協作貼文
   - 影片拍攝（場次、預計時數、拍攝日期）
   - 影片發布平台與廣告投放期間
9. **是否有肖像授權**：
   - **有**：需確認授權使用範圍（預設全媒體台澎金馬）、授權期間
   - **無**：跳過肖像授權條款
10. **額外條款/特別約定**：
    - 競品條款（例：Owndays 要求 KOL 合約期間不能代言其他眼鏡品牌）
    - 品牌特別約定（例：麥當勞額外的行為規範）
    - 對品牌的負評禁止條款
    - 其他客製條款

### Step 3：產生合約

收集完資訊後：

1. 先用 Read 工具讀取 docx skill（`/sessions/festive-zen-shannon/mnt/.claude/skills/docx/SKILL.md`）了解 docx 產生方式
2. 讀取本 skill 的 `references/contract-clauses.md` 取得所有條款的標準用語
3. 用 Node.js + docx 套件產生 .docx 檔案
4. 存檔至工作區的 `Outputs/Contract/` 資料夾
5. 檔名格式：`YYYYMMDD 專案名稱_合約類型.docx`（例：`20260326 農純鄉社群專案_社群合作協議.docx`）

### Step 4：驗證

1. 用 `python scripts/office/validate.py` 驗證 docx 檔案
2. 轉 PDF 截圖確認排版
3. 提供下載連結給使用者

## 合約結構對照

### A. 對客戶 — 社群合作協議

```
標題：「{專案名稱} x 只要有人社群合作協議」
前言（甲乙方資訊、合作期間）
第一條：權利義務（甲方 + 乙方）
第二條：智慧財產權歸屬與授權（著作財產權讓與甲方，乙方保留非營利使用權）
第三條：保密條款
第四條：違約處理
第五條：其他
[第六條：藝人/創作者合作]（選用，有 KOL 合作時加入）
簽名區（甲乙方資訊表格）
```

### B. 對客戶 — 顧問服務合約

```
標題：「{專案名稱} x 只要有人顧問服務合作協議」
前言（甲乙方資訊、合作期間）
第一條：權利義務（甲方 + 乙方，含驗收條款 15 工作天、乙方得非營利使用去識別化成果）
第二條：智慧財產權及所有權歸屬（著作人為乙方，IP 歸乙方）
第三條：保密條款
第四條：違約處理（含賠償上限為服務費用總額）
第五條：其他
簽名區（甲乙方資訊表格）
```

### C. 對藝人 KOL 合約

```
標題：「藝人/創作者合作合約」
前言（甲方=只要有人、乙方=經紀公司、甲方客戶=品牌、乙方藝人=KOL）
第 1 條：合作日期與內容（逐項列出所有合作項目）
第 2 條：合作費用與支付方式
第 3 條：著作權及所有權約定
[第 4 條：肖像授權]（選用）
第 N 條：非營利使用與報獎
[第 N+1 條：特別約定 / 防火條款]（選用）
第 N+2 條：確保無侵權及無違法
第 N+3 條：保護品牌商譽
第 N+4 條：賠償責任
第 N+5 條：保密義務
第 N+6 條：違約及損害賠償
第 N+7 條：其他
簽名區
```

## 重要的格式規範

### 文件格式
- 紙張大小：A4（因為是台灣法律文件）
- 字型：標題用粗體、內文用標準字重
- 字型建議：Arial 或 微軟正黑體（如系統有的話）
- 段落間距：適當留白，讓合約清楚易讀
- 條款編號：使用中文法律文件慣用格式（第一條、1.、(1)、a. 等層級）

### 簽名區格式
- 對客戶合約：使用表格，左欄甲方、右欄乙方，含公司名稱、統一編號、負責人、地址
- 對 KOL 合約：逐行列出甲方、乙方、乙方藝人的完整資訊

## docx 產生注意事項

產生合約 docx 時，遵守 docx skill 的所有規範，特別注意：
- 使用 `npm install -g docx` 安裝的 docx 套件
- A4 紙張尺寸：width: 11906, height: 16838
- 合約是正式法律文件，排版要整潔專業
- 條款子項目使用正確的編號格式（numbering config）
- 簽名區用 Table，設定好 column widths
- 產出後一定要用 `python scripts/office/validate.py` 驗證
